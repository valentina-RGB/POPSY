Introducción

El Servicio Nacional de Aprendizaje SENA Regional Antioquia, ofrece el programa de Tecnólogo en Análisis y Desarrollo de Software, entre las competencias que deben desempeñar los aprendices en el último trimestre de formación, encontramos la competencia “implantación del software” con el resultado de aprendizaje “Implantar el software de acuerdo con los niveles de servicio establecidos con el cliente”, el cual debe evidenciarse por medio del manual de usuario del software CREAMY SOFT.

El software CREAMY SOFT es el encargado de gestionar los procesos de configuración, usuarios, entradas y ventas; También, es el repositorio de las figuras que la empresa ha diseñado o que los clientes han solicitado.

Este manual fue diseñado con el objetivo de proporcionar al usuario la información necesaria para el correcto uso del software CREAMY SOFT, mediante una guía paso a paso sobre las diferentes funcionalidades que tiene el aplicativo.

4. Módulo de entradas

4.1 Gestión de insumos

El módulo de gestión de insumos está diseñado para que únicamente el administrador del sistema pueda administrar los insumos que sirven como materia prima para los productos. Este módulo proporciona herramientas para manejar y supervisar la información de cada insumo de manera efectiva. Las funcionalidades disponibles incluyen la creación, edición, y eliminación de insumos, así como la visualización de sus detalles, la gestión de su estado (activo/inactivo) y la adición de entradas de inventario.

4.1.1 Agregar un insumo

La funcionalidad de Agregar Insumos permite al administrador del sistema crear nuevos insumos. Al hacer clic en el botón "Agregar Insumo", se despliega un formulario con los siguientes campos: Descripción, Precio, Tipo de Insumo, Stock Máximo y Stock Mínimo.

Es importante destacar que, si el Tipo de Insumo seleccionado es Salsa o Helado, los campos de Stock Máximo y Stock Mínimo se deshabilitan, ya que estos tipos de insumos no requieren gestión de inventario. Una vez completados los campos obligatorios, el insumo se puede guardar para su uso en el sistema.

   

Paso 1.

4.2.2 Validación de Agregar Entrada de Insumo.

Al agregar una nueva entrada de insumo, el sistema realiza diversas validaciones para asegurar que la cantidad ingresada sea válida. En los siguientes casos, se mostrará un mensaje de error específico para guiar al usuario en la corrección de los datos:

1. Ingreso de Cantidad

  • Solo se permiten números enteros positivos.
  • Validación al ingresar la cantidad:
  • Si se ingresa un valor que no es un número entero positivo, el sistema mostrará el mensaje:
    "Por favor ingrese solo números enteros."
  • Al intentar enviar el formulario, si la cantidad ingresada no es un número entero o es menor o igual a cero, se mostrará el siguiente mensaje:
    "Por favor ingrese una cantidad válida y sin decimales."
  • Límite Máximo de Cantidad:
  • Si la cantidad ingresada excede el límite máximo de 120 unidades, el sistema mostrará el mensaje:
    "La cantidad supera el límite máximo permitido de 120."

2. Proceso de Envío de Datos

  • El formulario solo se enviará si la cantidad es válida (número entero positivo y dentro del límite máximo). Si alguna validación falla, el envío no se realizará y se mostrará el mensaje de error correspondiente.
  • Si la validación es exitosa, se enviará una solicitud para agregar la entrada de insumo, y el usuario verá una notificación de éxito:
  • "La entrada del insumo se ha agregado exitosamente."

3. Manejo de Errores

  • Si hay algún problema con la solicitud (por ejemplo, error en la conexión o al procesar la entrada), el sistema notificará al usuario:
    "No se pudo agregar la entrada del insumo. Por favor, intente nuevamente."

Glosario

Validación: la validación se refiere al proceso de comprobar que los datos ingresados en la aplicación o sistema son correctos y cumplen con los requisitos establecidos. La validación es una parte importante del proceso de entrada de datos, ya que asegura que los datos sean precisos y útiles.

Usuario: en el contexto de la informática, un usuario se refiere a una persona o entidad que utiliza un sistema o aplicación. En otras palabras, el usuario es la persona que interactúa con la tecnología para realizar tareas específicas.

Rol: en el contexto de la informática, un rol se refiere a un conjunto de permisos o derechos que se le otorgan a un usuario dentro de un sistema o aplicación. Los roles son utilizados para definir las diferentes funciones que pueden realizar los usuarios en el sistema.

Permiso: Un permiso es fundamental para la seguridad y el control de acceso dentro de un sistema informático, permitiendo que los administradores y usuarios gestionen de manera eficiente las capacidades de interacción con recursos digitales.

Producto: Es cualquier artículo que se ofrece a los usuarios para su compra. Su presentación efectiva y clara es crucial para atraer a los usuarios y facilitar las transacciones.

Categoría: Una categoría de producto organiza y agrupa los productos de manera lógica, permitiendo a los usuarios encontrar más fácilmente lo que buscan. Es una herramienta esencial para mejorar la experiencia de navegación, la organización del sitio web y la eficiencia del proceso de compra, lo cual puede generar mayores conversiones y ventas.

Compra: Es el proceso integral que permite a los administradores seleccionar, pagar y recibir productos o servicios para así llevar una correcta contabilidad de las compras generadas.

Entrada:  Permite al administrador incrementar el stock de un insumo específico, siendo que no tenga que volver a registrar un insumo.

Dashboard: En el contexto de una página web, un dashboard (o tablero de control) es una interfaz visual que proporciona un resumen y acceso rápido a la información clave y las métricas de un sistema o plataforma. Generalmente, se utiliza en aplicaciones administrativas o de análisis, mostrando datos relevantes en tiempo real, indicadores de rendimiento, y herramientas de gestión.
